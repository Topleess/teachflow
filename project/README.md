# TeachFlow · Project OS

Это единый операционный контур проекта. Начинать работу нужно с `STATUS.md`, затем смотреть активный фокус в `07-operations/NEXT.md`.

## Структура

| Папка | Назначение | Главный результат |
|---|---|---|
| `00-inbox/` | Неразобранные идеи, ссылки, файлы и заметки | Пустой inbox после регулярного разбора |
| `01-strategy/` | Цель, аудитория, принципы и решения | Понятно, что строим и почему |
| `02-product/` | Возможности, готовность, сценарии и ограничения | Проверяемое продуктовое определение |
| `03-research/` | Интервью, реальные уроки и выводы | Факты вместо придуманных примеров |
| `04-site/` | Контент, доказательства и QA лендинга | Понятная страница по холодной ссылке |
| `05-sales/` | One-pager, deck и сценарии контакта | Материалы, согласованные с продуктом |
| `06-leads/` | База контактов и история касаний | Управляемая воронка без спама |
| `07-operations/` | План, статус, риски, ритм и следующие действия | Проект не разваливается на отдельные треки |
| `99-archive/` | Отвергнутые и устаревшие версии | История сохраняется, активный слой остаётся чистым |

## Правила

1. Один источник истины для каждого типа данных.
2. Будущая возможность не называется готовой функцией.
3. Любой пример на сайте опирается на реальный кейс или явно помечен как концепция.
4. Исходящие обращения остановлены, пока продуктовая история не проходит cold-link test.
5. Каждое решение фиксируется в `01-strategy/DECISIONS.md`.
6. Каждая рабочая сессия заканчивается обновлением `STATUS.md` и `07-operations/NEXT.md`.
