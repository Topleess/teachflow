# TeachFlow · handoff

## What is ready

A final responsive landing page combining the premium editorial composition of concept 1 with the collaborative lesson canvas of concept 3.

The page includes:
- clear product positioning;
- visual lesson scenario;
- preparation → live lesson → saved result → homework flow;
- audiences;
- pilot CTA and accessible modal form;
- mobile transformation and reduced-motion support.

## Decisions intentionally left to the receiving team

1. **Form destination:** CRM, email, Telegram bot or custom API.
2. **Public domain and hosting.**
3. **Real contact details and responsible legal entity.**
4. **Privacy-policy URL and consent wording.**
5. **Product truth:** confirm which demonstrated features exist in the prototype and adjust copy if needed.
6. **Analytics:** add only after choosing a provider and consent policy.

## Publication checklist

- [ ] Connect form and verify a real submission receipt.
- [ ] Add privacy-policy link and consent text.
- [ ] Add favicon and social preview image.
- [ ] Confirm owner/contact information.
- [ ] Check all copy against the current product scope.
- [ ] Deploy and test the live domain on mobile and desktop.
- [ ] Run accessibility and performance checks against the deployed origin.

## Important limitation

The current form deliberately does not transmit personal data. It shows a transparent demonstration message. Do not replace that message with a false “sent” confirmation until the backend is connected and read-back verified.
